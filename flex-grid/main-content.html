Ok, but what's the difficulty her?

Just try to get all points in the list done at once, good luck ;-)

Just chocking, i will try to explain it. You can get most of the points easily with display grid, but there is no chance to get items in a grid centered. For example think about a 3 columns grid with 4 items. The fourth item will not center, there is no way with display grid, expect a really hacky solution with nth-last-child, last-child and doubling the columns and so on.

So your best option is flexbox, but it's also hard to achieve all the points, without writing multiple media-queries for different screen-widths over and over again.

Other Questions you may ask ;-)

How did you do that?

I created a formula with calc, that calculates the size of the items base on items min-width and fg-gapwidth. Items number and gap width are css variables. These variables are set in multiple container-queries, so that we can use it in every nested context, no matter how wide the parent element is.

Why did you do that?

  • Because I wanted to solve that problem
  • Because I like challenges like that
  • Because I like build stuff with pure css

Why not using tailwind or similar?

  • Because it's way to much overkill and would kill the pagespeed because of features we never use.
  • I just needed to solve that one problem and this for less than 3kb gives me the best price-performance ratio I can imagine.

Why not using sass?

  • I wanted to build something anyone can use, without having to know how to implement sass in his own workflow. It's just copy&paste the file in your project and you are ready.
  • I am also working on a fun project, it's a kind of minimal cms without a database, composer and so on. It is something anyone can just copy&paste on his server an just start using it. This also means the cms just uses html, css and js for frontend, nothing else.
  • SASS in the end compiles to css, so I would get no size benefits of it
